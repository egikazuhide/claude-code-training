## トークスクリプト

### 【オープニング】（約1分）

「前回は1ページのランディングページを作りました。今回は複数ページで構成された社内サイトを作ります。

社内のマニュアル・規程集・FAQ集などを1か所にまとめたサイトを作れます。インターネット接続不要でローカルに保存できるので、社内配布にも使えます。」

---

### 【本編①：複数ページサイトの構成】（約2分）

「複数ページのWebサイトは、フォルダに複数のHTMLファイルを格納する形で作ります。

例えば社内マニュアルサイトなら以下の構成です。

```
manual_site/
├── index.html         ← トップページ
├── basic/
│   ├── login.html     ← ログイン手順
│   └── menu.html      ← メニュー操作
├── advanced/
│   ├── report.html    ← レポート作成
│   └── export.html    ← データ出力
└── css/
    └── style.css      ← 全ページ共通スタイル
```

共通のナビゲーションメニューを作ることで、各ページを行き来できます。

Claude Codeにフォルダ構成と全ページのHTML生成を一度に頼めます。」

---

### 【本編②：実演：社内マニュアルサイトを作る】（約5分）

「*【画面収録開始】*

チャット欄に入力します。

『以下の構成で社内システムのマニュアルサイトをHTMLで作成してください。

フォルダ名：manual_site

ページ構成：
・index.html：トップページ（目次・各マニュアルへのリンク）
・login.html：ログイン・ログアウト手順
・basic.html：基本操作マニュアル
・faq.html：よくある質問

共通仕様：
・全ページ共通のヘッダー（サイトタイトル・ナビゲーション）
・全ページ共通のフッター（作成日・問い合わせ先）
・共通CSSファイル（style.css）を使用
・左サイドバーにページ内目次を表示
・スマートフォン対応

デザイン：
・シンプルなビジネスサイトの雰囲気
・メインカラー：濃紺（#1E3A5F）
・背景：白

各ページのコンテンツは「サンプルテキスト」でOK』

---

複数のファイルが生成されました。ファイルツリーを確認するとmanual_siteフォルダに構成通りにファイルが作られています。

index.htmlをブラウザで開きます。ヘッダー・ナビゲーション・フッターが統一されたサイトが表示されました。

ナビゲーションをクリックして各ページを確認します。全ページで同じデザインが維持されています。

*【画面収録終了】*」

---

## 操作・画面の流れ（詳細版）

### 📋 収録前準備チェックリスト
- [ ] 社内マニュアルテキスト（3つのセクション分）をデスクトップに用意
- [ ] Claude Codeを起動し空のプロジェクトを開く
- [ ] ブラウザを起動しておく
- [ ] 画面録画を開始する（音声テスト済み）
- [ ] 通知をオフにする

### 🎬 収録手順（9分完全版）

#### 【オープニング】 0:00〜1:00
- 「今日は複数ページのWebサイトをまとめて生成します」

#### 【本編①：複数ページサイトの構成】 1:00〜3:00
- 「index.html（トップ）・manual.html（マニュアル）・faq.html（FAQ）の3ページ構成を作ります。ナビゲーションで行き来できます」

#### 【本編②：実演 社内マニュアルサイト作成】 3:00〜8:00
1. **3:00〜3:20** テキストファイルをドラッグ&ドロップ
2. **3:20〜3:50** 「このテキストをもとに、トップページ・業務マニュアル・FAQの3ページからなる社内サイトを生成してください。ナビゲーションで各ページに遷移できるようにしてください」と入力してEnter
3. **3:50〜5:00** 承認でYes → 3つのHTMLファイルが生成される → サイドバーで確認
4. **5:00〜6:00** ブラウザでindex.htmlを開く → ナビゲーションを映す
5. **6:00〜7:00** ナビゲーションのリンクをクリックして各ページに移動 → 映す
6. **7:00〜8:00** 「ナビゲーションのデザインを整えてください」と修正 → 確認

#### 【クロージング】 8:00〜9:00
- 「次回はブラウザで確認しながらデザインを調整します」と締める


### ⚠️ 収録時の注意点
- 各操作後は2〜3秒間画面を止めて視聴者に確認の時間を与える
- ファイルが生成された瞬間・ブラウザで開いた瞬間など成果が見えるシーンはゆっくり映す
- 承認ダイアログは必ず3ボタンが見えるよう拡大表示してからYesをクリック
