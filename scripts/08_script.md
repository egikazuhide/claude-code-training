## トークスクリプト

### 【オープニング】（約1分）

「前回はGitでの変更管理、しっかり学びましたね。今回のテーマは——『CLAUDE.md（クロードエムディー）』という特別なファイルです。

Claude Codeを使っていると、毎回同じことを説明しなきゃいけない場面、出てきます。『このアプリはどういうアプリか』『使っているツールは何か』『ファイルはどこにあるか』——こういう前提知識、毎回言うのめんどくさいですよね。

CLAUDE.mdという特別なファイル（プロジェクトの説明書として、Claude Codeが自動で読み込んでくれる設定ファイル）を作ると——Claude Codeがプロジェクトを開くたびに自動でそれを読んで、前提を理解した状態で作業を始めてくれるんです。もう毎回説明しなくていい。最高でしょ。」

---

### 【本編①：CLAUDE.mdとは何か】（約2分）

「CLAUDE.mdとは、プロジェクトフォルダの中に作る特別なテキストファイルです。

このファイルに書かれた内容を、Claude Codeはプロジェクトを開くたびに最初に読みます。いわば『このプロジェクトの説明書』ですね。

---

たとえばですよ。あなたが新しいアルバイトスタッフを採用したとします。毎日出勤のたびに『うちの店はこういう店で、レジはここにあって、商品はここにある』って説明するの、大変じゃないですか。

そこで最初に業務マニュアルを渡しますよね。スタッフはそれを読めば、毎回同じ説明をしなくても仕事を進められる。

CLAUDE.mdは、まさにその業務マニュアルです。一度作れば、Claude Codeは毎回それを読んで、プロジェクトの前提を把握した状態で作業してくれます。」

---

### 【本編②：CLAUDE.mdに何を書くか】（約3分）

「では、CLAUDE.mdに何を書けばいいのか。プロジェクトによって変わりますが、基本的にはこの4項目を書くと効果的です。

---

**①プロジェクトの概要**

このプロジェクトが何を作るものなのかを1〜3行で書きます。

例：
```
## プロジェクト概要
動画ファイルを読み込んで、画像つきステップマニュアルを
自動生成するデスクトップアプリです。
```

---

**②技術スタック（使っているツール）**

どのプログラミング言語・フレームワーク・ライブラリを使っているかを書きます。Claude Codeが適切な方法で作業するために必要な情報です。

例：
```
## 使用技術
- Python 3.11
- ffmpeg（動画処理）
- Claude API（画像認識）
- tkinter（画面作成）
```

---

**③フォルダ構成**

主要なフォルダとファイルの説明を書きます。

例：
```
## フォルダ構成
- /src：アプリの本体コード
- /output：生成されたマニュアルの保存先
- /temp：作業用の一時ファイル
```

---

**④注意事項・ルール**

Claude Codeに守ってほしいルールがあれば書きます。

例：
```
## 注意事項
- コードのコメントは必ず日本語で書いてください
- 変更後は必ずGitでコミットしてください
```

---

CLAUDE.mdは『Markdown（マークダウン）』形式で書きます。Markdownとは、特定の記号を使って見出しや箇条書きを表現する書き方のことです。難しくないので、実演で確認していきましょう。」

---

### 【本編③：実演】（約2分）

「*【画面収録開始】*

Claude Codeのチャット欄に、次のように入力します。

『このプロジェクトのCLAUDE.mdを作成してください。内容は以下です。

プロジェクト概要：動画ファイルを読み込んで画像つきマニュアルを生成するアプリ
使用技術：Python、ffmpeg、Claude API
注意事項：コメントは日本語で書く・変更後は必ずGitでコミットする』

承認して実行すると、プロジェクトフォルダにCLAUDE.mdが作成されます。出来上がりました。

---

作成されたCLAUDE.mdの中身を、ファイルツリーからクリックして確認します。内容がきれいに整形されてファイルに書かれています。

次回からClaude Codeは、このファイルを読んでプロジェクトの前提を理解した状態で作業を始めてくれます。もう新人教育、終わりました。

*【画面収録終了】*」
