## トークスクリプト

### 【オープニング】（約1分）

「前回はGitでの変更管理を学びました。今回は『CLAUDE.md』という特別なファイルについて学びます。

Claude Codeを使っていると、毎回同じことを説明しなければならない場面が出てきます。たとえば『このアプリはどういうアプリか』『使っているツールは何か』『ファイルはどこにあるか』といった前提知識です。

CLAUDE.mdを作ると、Claude Codeがプロジェクトを開くたびに自動でそのファイルを読んで、前提を理解した状態で作業を始めてくれます。」

---

### 【本編①：CLAUDE.mdとは何か】（約2分）

「CLAUDE.mdとは、プロジェクトフォルダの中に作る特別なテキストファイルです。

このファイルに書かれた内容を、Claude Codeはプロジェクトを開くたびに最初に読みます。いわば『このプロジェクトの説明書』です。

---

たとえば、あなたが新しいアルバイトスタッフを採用したとします。毎日出勤のたびに『うちの店はこういう店で、レジはここにあって、商品はここにある』と説明するのは大変ですよね。

そこで最初に業務マニュアルを渡します。スタッフはそれを読めば、毎回同じ説明をしなくても仕事を進められます。

CLAUDE.mdはその業務マニュアルに相当します。一度作れば、Claude Codeは毎回それを読んでプロジェクトの前提を把握した状態で作業してくれます。」

---

### 【本編②：CLAUDE.mdに何を書くか】（約3分）

「CLAUDE.mdに書く内容は、プロジェクトによって異なりますが、基本的に以下の項目を書くと効果的です。

---

**①プロジェクトの概要**

このプロジェクトが何を作るものなのかを1〜3行で書きます。

例：
```
## プロジェクト概要
動画ファイルを読み込んで、画像つきステップマニュアルを
自動生成するデスクトップアプリです。
```

---

**②技術スタック（使っているツール）**

どのプログラミング言語・フレームワーク・ライブラリを使っているかを書きます。Claude Codeが適切な方法で作業するために必要な情報です。

例：
```
## 使用技術
- Python 3.11
- ffmpeg（動画処理）
- Claude API（画像認識）
- tkinter（画面作成）
```

---

**③フォルダ構成**

主要なフォルダとファイルの説明を書きます。

例：
```
## フォルダ構成
- /src：アプリの本体コード
- /output：生成されたマニュアルの保存先
- /temp：作業用の一時ファイル
```

---

**④注意事項・ルール**

Claude Codeに守ってほしいルールがあれば書きます。

例：
```
## 注意事項
- コードのコメントは必ず日本語で書いてください
- 変更後は必ずGitでコミットしてください
```

---

CLAUDE.mdはMarkdown形式で書きます。Markdownとは、特定の記号を使って見出しや箇条書きを表現する書き方です。難しくないので実演で確認しましょう。」

---

### 【本編③：実演】（約2分）

「*【画面収録開始】*

Claude Codeのチャット欄に次のように入力します。

『このプロジェクトのCLAUDE.mdを作成してください。内容は以下です。

プロジェクト概要：動画ファイルを読み込んで画像つきマニュアルを生成するアプリ
使用技術：Python、ffmpeg、Claude API
注意事項：コメントは日本語で書く・変更後は必ずGitでコミットする』

承認して実行すると、プロジェクトフォルダにCLAUDE.mdが作成されます。

---

作成されたCLAUDE.mdの中身をファイルツリーからクリックして確認します。内容が整形されてファイルに書かれています。

次回からClaude Codeはこのファイルを読んでプロジェクトの前提を理解した状態で作業を始めます。

*【画面収録終了】*」
