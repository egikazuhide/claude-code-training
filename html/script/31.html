【オープニング】（約1分）


「アプリを作っていると必ずエラーが出ます。初心者の方がClaude Codeを使うとき、エラーが出るたびにどう対処すればいいかわからず止まってしまうことがよくあります。


この動画を見ると、エラーが出ても怖くなくなります。エラーへの対処法はシンプルで、エラーのメッセージをそのままClaude Codeに渡すだけです。」



【本編①：エラーを恐れない】（約2分）


「エラーは失敗ではありません。『ここが問題ですよ』というシステムからのメッセージです。


プロのエンジニアでも毎日エラーと向き合っています。エラーが出ることは当たり前のことで、問題はエラーを解決できるかどうかです。



Claude Codeが登場する前は、エラーが出たら自分でエラーの意味を調べて、解決策を考えて、コードを修正するという工程が必要でした。これが初心者には難しかった。


Claude Codeがあれば、エラーのメッセージをそのままコピーして貼り付けるだけで、原因と解決策を説明してくれて、修正まで行ってくれます。


エラーが出たら『チャンス』だと思ってください。修正方法を学べる機会です。」



【本編②：エラーの種類と読み方】（約3分）


「エラーには様々な種類がありますが、よく見るパターンを3つ覚えておきましょう。



パターン1：モジュールが見つからないエラー


『ModuleNotFoundError: No module named ○○』


日本語訳：『○○というモジュール（ライブラリ）が見つかりません』


原因：必要なライブラリがインストールされていない


対処法：『このエラーが出ました。解決してください』とClaude Codeに渡す



パターン2：ファイルが見つからないエラー


『FileNotFoundError: No such file or directory: ○○』


日本語訳：『○○というファイルやフォルダが存在しません』


原因：指定したパスにファイルがない。パスの記述ミスが多い。



パターン3：構文エラー


『SyntaxError: invalid syntax』


日本語訳：『プログラムの書き方が間違っています』


原因：コードの書き方に誤りがある。Claude Codeが生成したコードでは稀だが起きることがある。



エラーを見たとき読む必要があるのは最後の数行だけです。Pythonのエラーメッセージは上から下に向かってエラーの経路が書かれていて、最後の行が実際のエラー内容です。」



【本編③：実演：エラーをClaude Codeに渡して直す】（約3分）


「【画面収録開始】


実際にエラーが出ている状況を作ります。


わざとエラーが出るような状況を再現します。requirements.txtからライブラリを一つ削除して、アプリを起動します。


出力ログエリアに赤い文字でエラーメッセージが表示されました。



このエラーメッセージをすべて選択してコピーします。Ctrl+A（Mac:Command+A）で全選択してCtrl+C（Mac:Command+C）でコピーします。


チャット欄に貼り付けて、一言添えます。


『以下のエラーが出ました。原因を教えてそのまま修正してください。


（エラーメッセージを貼り付け）』



Claude Codeがエラーの原因を説明して修正してくれます。ライブラリが不足していたためインストールを行ってくれます。


修正後にもう一度アプリを起動して、エラーが解消されたことを確認します。


【画面収録終了】」
