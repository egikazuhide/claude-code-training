家を建てるとき、まず基礎工事をしますよね。アプリ開発も同じで、まず『土台』を作ってから各機能を積み上げていきます。」



【本編①：フォルダ構成とは何か】（約2分）


「フォルダ構成とは、アプリを構成するファイルをどのフォルダに入れるかの整理の仕方です。


なぜフォルダ構成が重要かというと、ファイルが増えたときに探しやすくするためです。整理されていないと、どこに何があるかわからなくなります。



今回作るアプリのフォルダ構成のイメージは以下です。



manual_creator/        ← プロジェクトのルートフォルダ
├── src/               ← アプリの本体コード
│   ├── main.py        ← アプリの起動ファイル
│   ├── extractor.py   ← 動画から画像を切り出す処理
│   └── generator.py   ← AIで説明文を生成する処理
├── output/            ← 生成したマニュアルの保存先
├── temp/              ← 作業中の一時ファイル
├── CLAUDE.md          ← プロジェクトの説明書
└── requirements.txt   ← 使うライブラリの一覧

このような構成を、Claude Codeに一発で作ってもらいます。」



【本編②：実演：Claude Codeにフォルダ構成を作らせる】（約4分）


「【画面収録開始】


まず作業フォルダを作ります。デスクトップに『manual_creator』というフォルダを作成します。


次にClaude Codeでこのフォルダをプロジェクトとして開きます。



チャット欄に以下の指示を入力します。


『以下の仕様でマニュアル自動生成アプリのプロジェクトを作成してください。


概要：動画ファイルを読み込んで、ffmpegでスクリーンショットを切り出し、Claude APIで各画像の説明文を生成して、画像つきステップマニュアルとして出力するデスクトップアプリです。


作成してほしいもの：

・src/main.py（アプリの起動ファイル・画面の骨格）

・src/extractor.py（ffmpegで動画から画像を切り出す処理）

・src/generator.py（Claude APIで説明文を生成する処理）

・output/フォルダ（空でOK）

・temp/フォルダ（空でOK）

・requirements.txt（必要なライブラリの一覧）

・CLAUDE.md（このプロジェクトの説明）


使用言語：Python

画面作成：tkinter（追加インストール不要）』



送信すると、Claude Codeが各ファイルを作成し始めます。承認ダイアログが出るたびに『Yes』をクリックします。


しばらくするとすべてのファイルが作成されます。ファイルツリーで確認してください。


【画面収録終了】」



【本編③：生成されたファイルをエディタで確認する】（約2分）


「【画面収録開始】


生成されたファイルの中身を確認します。


ファイルツリーから『src/main.py』をクリックします。コードが表示されますが、内容を全部理解する必要はありません。


注目してほしいのは2点です。


1点目：ファイルの先頭に説明コメントが書かれているか確認します（日本語で書かれているはず）


2点目：ファイルが適切な場所に作られているか確認します


『requirements.txt』を開くと、このアプリが使うライブラリの一覧が書かれています。これをClaude Codeにインストールさせます。


『requirements.txtに書かれているライブラリをインストールしてください』


インストールが完了したら土台の準備が完了です。


【画面収録終了】」
