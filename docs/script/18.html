第30回｜見栄えのするランディングページを作る


尺：10分 / Part8

ゴール：デザインの整ったWebページを指示だけで作れる

収録スタイル：操作画面を映しながら音声解説



構成


ブロック内容時間
オープニング前回の復習・この動画の目的1分
本編①ランディングページとは1分
本編②実演：商品・サービス紹介ページを作る7分
クロージングまとめ・次回予告1分


トークスクリプト


【オープニング】（約1分）


「前回シンプルなHTMLページを作りました。今回は、より完成度の高いランディングページを作ります。


ランディングページとは、商品やサービスを紹介する1ページ完結のWebページです。会社のサービス紹介・イベント告知・商品紹介などに使えます。」



【本編①：ランディングページとは】（約1分）


「ランディングページの構成要素を説明します。


一般的な構成：


  • ヒーローセクション（大きなキャッチコピー・イメージ）
  • 特徴・メリットの紹介
  • 使い方・フロー説明
  • お客様の声（テスティモニアル）
  • よくある質問
  • お問い合わせ・CTA（行動促進）

    • これらのセクションをClaude Codeに指示すると、一気に作ってくれます。」



    【本編②：実演：商品・サービス紹介ページを作る】（約7分）


    「【画面収録開始】


    今回のアプリ（マニュアル自動生成アプリ）のランディングページを作ってみましょう。


    チャット欄に入力します。


    『「マニュアル自動生成アプリ」のランディングページをHTMLで作成してください。


    ページ構成：

    ① ヒーローセクション

    キャッチコピー：「動画を渡すだけでマニュアルが完成」

    サブコピー：「業務手順書作成の時間を90%削減」

    CTAボタン：「無料でダウンロードする」（ボタンの動きはなくてOK）


    ② 3つの特徴

    「5秒ごとに画像を自動切り出し」

    「AIが操作説明文を自動生成」

    「Word・PDFで即出力」

    各特徴にアイコン（絵文字でOK）と短い説明文をつける


    ③ 使い方の流れ（3ステップ）

    Step1：動画をドロップ

    Step2：AIが自動処理

    Step3：マニュアル完成・出力


    ④ FAQセクション（3つの質問と回答）


    デザイン要件：

    ・メインカラー：青（#3B82F6）

    ・白背景・モダンでシンプルなデザイン

    ・スマートフォン対応

    ・ファイル名：landing.html』



    処理が完了したらブラウザで開きます。


    プロが作ったような見栄えのあるランディングページが完成しました。スクロールしながら各セクションを確認します。


    ヒーローセクション・特徴紹介・使い方の流れ・FAQが綺麗に並んでいます。


    スマートフォンサイズで確認します。ブラウザの幅を狭くすると、1列レイアウトに切り替わります。



    一部修正をします。


    『ヒーローセクションの背景に薄いグラデーション（白→薄い青）を追加してください』


    修正が即座に反映されました。


    【画面収録終了】」



    【クロージング】（約1分）


    「まとめます。


  • ランディングページはセクション構成を指示するだけで作れる
  • デザイン要件（カラー・スタイル）を具体的に指示することが重要
  • 修正も一言指示で即座に反映できる

    • 次回は複数ページの社内サイトを作ります。


    お疲れさまでした。」



    操作・画面の流れ（詳細版）


    📋 収録前準備

  • [ ] 商品・サービス名と簡単な説明文をテキストで用意
  • [ ] Claude Codeを起動し空のプロジェクトを開く

    • 🎬 収録手順


    #### 【オープニング】（0:00〜1:00）

  • 「今日はデザインのきれいなWebページを作ります」と話す

    • #### 【本編①：ランディングページとは】（1:00〜2:00）

  • 「製品・サービスを紹介する縦長のページです」と口頭で簡潔に説明

    • #### 【本編②：実演 商品・サービス紹介ページを作る】（2:00〜9:00）

  • 用意したサービス説明テキストをドラッグ&ドロップ
  • 「このサービス情報を使って、プロらしいランディングページのHTMLを作ってください。ヒーロー画像エリア・特徴3点・CTAボタンを含めてください」と入力
  • 承認で Yes → 複数ファイル（HTML+CSS）が生成される
  • ブラウザで開く → デザインを映す
  • 「もう少し余白を広くしてください」「フォントをゴシック体にしてください」など2〜3回修正を加える
  • 最終的なページを映す

    • #### 【クロージング】（9:00〜10:00）

  • 「このページをサーバーにあげれば公開できます」と話して締める

    • ⚠️ 収録時の注意点

  • ブラウザで開いたページをスクロールして全体を見せること
